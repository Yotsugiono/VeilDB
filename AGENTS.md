# AGENTS.md

本文件用于约束本仓库的默认开发流程，目标是减少重复沟通、减少返工，并让改动和当前项目结构保持一致。

如果本文件与仓库中的脚本、工作流、代码现状不一致，以实际可执行内容为准，并在相关改动中顺手修正文档，避免规则继续漂移。

## 技术栈与目录结构
- **核心服务端 / 加密数据库**：`EncDB/System_high`，C/C++ + Intel SGX，包含 Enclave 代码、RPC/TCP 服务端、客户端和 benchmark，主要可执行文件为 `encdb_server`、`encdb_client`、`encdb_benchmark`。
- **数据与持久化目录**：`EncDB/data` 存放示例数据和生成脚本；`EncDB/Databases` 存放运行时数据库、密文文档、bitmap、catalog 等持久化文件。
- **Web 网关 / 接口层**：`web/gateway`，Python + FastAPI，负责提供 HTTP API，并桥接到 EncDB 的 TCP/RPC 服务。
- **前端界面**：`web/frontend`，静态 HTML/CSS/JavaScript，通过 FastAPI 挂载到 `/static/index.html`。
- **Web 依赖与工具**：`web/requirements.txt` 记录 Python 依赖；`web/scripts` 存放调试脚本。
- **项目文档**：根目录 `README.md` 为统一说明文档；本文件 `AGENTS.md` 约束仓库开发流程。

## 工作流程（必须遵守）
1. **先读后写**：在创建或修改任何文件之前，必须先读取目标文件及相关的说明文档（如 README、接口文档等），确保理解现有上下文。
2. **最小修改原则**：仅修改完成当前任务所必需的代码，不随意格式化无关代码，不重构未涉及的部分。
3. **重大修改确认机制**：任何涉及以下情况的“大修改”，必须在写代码前向我说明计划并等待我的确认：
   - 删除或重命名现有文件、函数、数据库表。
   - 修改超过 50 行以上的核心逻辑代码。
   - 新增外部依赖（如 requirements.txt / package.json 的变动）。
   - 修改 API 接口的入参或返回值结构。
4. 发现中文乱码不要擅自修改，告诉我位置，我自己来改。
